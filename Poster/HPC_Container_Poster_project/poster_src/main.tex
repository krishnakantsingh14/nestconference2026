% ================= ROW 1 : INTRODUCTION =================
\begin{block}{1.\ \ Introduction --- reproducibility lives in the dependency graph}
\large
Reproducibility in computational neuroscience depends not only on the simulator but on its \emph{entire dependency
graph}. \tl{NEST\,3.9} carries \hl{72 transitive dependencies} (e.g., OpenMPI, Boost, jemalloc, Python). Deploying such
workflows across HPC is hindered by site-specific dependencies, heterogeneous OS, and microarchitecture-specific
compiler optimisations. The \tl{EBRAINS Software Distribution (ESD)} tracks dependencies rigidly via \emph{Spack},
yet synchronising identical environments from ESD cloud deployments onto specific HPC hosts still leads to
\hl{dependency hell}.
\medskip

\centering\textbf{\large We present a \hl{zero-install} approach:\\portable, HPC-enabled \tl{Apptainer} images that
reach \hl{near-bare-metal} performance --- and ask: \emph{how much does portability cost?}}
\end{block}

% ================= ROW 2 : OUR APPROACH =================
\begin{block}{2.\ \ Our approach --- HPC-enabled, streaming containers}
\begin{columns}[T]
  \begin{column}{0.40\textwidth}
    \large
    \heading{Runtime Portability --- PMIx bridges container-host MPI}
    The image encapsulates \tl{OpenMPI\,5.x, UCX, verbs} and a \hl{PMIx client}. At runtime the client connects to
    the \hl{host-side PMIx server} exposed by SLURM, bridging container MPI to the host fabric, process manager and
    scheduler simultaneously.
    \medskip

    \heading{Compile-time Portability --- Multi-architecture Builds}
    \tl{QEMU} user-mode emulation inside Apptainer produces \hl{architecture-optimised} builds
    (AVX2 \& AVX-512, Intel/AMD) from a single host.
    \medskip

    \heading{Toward ``Software Streaming''}
    The dependency stack moves onto \hl{CernVM-FS}: the image ships only the application; the EESSI stack is
    \emph{streamed on demand}, so one thin image runs across EuroHPC sites.
  \end{column}
  \begin{column}{0.585\textwidth}
    \begin{columns}[T]

% ================= ROW 5 : CONCLUSION + REFERENCES =================
\begin{columns}[T]
  \begin{column}{0.60\textwidth}
    \begin{block}{5.\ \ Conclusion}
    \large
    \begin{itemize}
      \vitem \hl{Portability is essentially free:} containerised MPI matches native at the fabric (OSU, NCCL)
      and application (NEST) level.
      \vitem \tl{Build once, run anywhere:} PMIx wire-up + QEMU multi-arch + CernVM-FS streaming give one thin,
      EuroHPC-portable image --- a practical, zero-install path for the ESD toward on-demand HPC deployment.
    \end{itemize}
    \end{block}
